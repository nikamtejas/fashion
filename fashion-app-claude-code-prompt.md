## MASTER CONTEXT (save as CLAUDE.md)

You are building LuxeLoom, a production-grade fashion e-commerce platform for the Indian market with a modern, unique, premium UI. The frontend is Next.js 14+ with the App Router, TypeScript, Tailwind CSS, Framer Motion for animations, and heavily customized shadcn/ui base components, with Zustand for client state. The backend uses Next.js API routes with Mongoose as the ODM. The database is MongoDB Atlas (live cloud cluster) — connect via `MONGODB_URI` in the environment, use a cached connection pattern suitable for serverless, define proper Mongoose schemas with indexes on frequently queried fields (product slug, category, order status, user email, pincode), and never store secrets in code. All product and generated images are stored in Cloudinary — use the official Cloudinary Node SDK, upload via signed server-side uploads, store the `public_id` and `secure_url` in MongoDB, organize assets into folders per product, and always serve images through Cloudinary transformation URLs (f_auto, q_auto, responsive widths) for performance. Authentication is NextAuth.js with email OTP and Google login, with role-based access for CUSTOMER and ADMIN. PDF generation uses pdfkit or @react-pdf/renderer. Maps use Leaflet with OpenStreetMap.

The design language is an editorial fashion-magazine aesthetic: a large serif display font such as Fraunces or Playfair Display paired with Inter, an ivory background (#FAF7F2), ink black (#141414), a burnt sienna accent (#C15B3C), muted sage for success states, generous whitespace, oversized product imagery, asymmetric editorial grids rather than uniform cards, micro-interactions everywhere (hover reveals, image zoom, magnetic buttons, smooth page transitions, skeleton loaders), a warm charcoal dark mode, and a fully responsive mobile-first PWA-installable build.

External integrations are Google Gemini 2.x image API (`GEMINI_API_KEY`) for AI photo generation, Razorpay (`RAZORPAY_KEY_ID`, `RAZORPAY_KEY_SECRET`) for online payments, Snapmint (`SNAPMINT_MERCHANT_ID`, `SNAPMINT_API_KEY`) for EMI, Blue Dart / DHL India (`BLUEDART_LICENSE_KEY`, `BLUEDART_LOGIN_ID`) for shipping, and Cloudinary (`CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`). Every external call goes through a service layer in `/lib/integrations/` with timeouts, retries, and a global `INTEGRATIONS_MOCK=true` flag that returns realistic mock responses so the entire app works end-to-end in development without real keys. Provide a complete `.env.example`.

Ground rules: every milestone must end with the app running via `npm run dev` with zero TypeScript errors; seed the Atlas database with 20 realistic fashion products across Men, Women, Accessories and Footwear plus 3 store locations with real Indian lat/lng and pincodes; commit after each milestone with a descriptive message; always show me a short plan before implementing.
